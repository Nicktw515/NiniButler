# 尼尼管家公｜旅行記帳分帳工具

單一 HTML 檔的純前端旅行記帳 App，資料存在你自己的 Google Sheet 裡，透過 Google Apps Script 當後端 API。沒有登入、沒有帳號，離線也能先用。

## 檔案說明

| 檔案 | 用途 |
|---|---|
| `index.html` | 整個 App，直接部署到 GitHub Pages 或用瀏覽器打開即可用（離線模式） |
| `Code.gs` | Google Apps Script 後端程式碼，負責讀寫 Google Sheet |
| `manifest.json` / `icon-180.png` / `icon-512.png` | 讓手機瀏覽器可以「加入主畫面」，變成全螢幕的 App 圖示體驗，記得跟 `index.html` 放在同一個資料夾 |
| `husky-mascot.svg` | 尼尼的獨立形象檔（跟 App 裡用的是同一版），可以拿去做貼圖、App 圖示或其他地方 |

## 部署步驟

### 1. 建立你的旅行 Google Sheet
開一個新的 Google Sheet，隨便命名，例如「東京旅行帳本」。

### 2. 掛上後端程式
在該 Sheet 裡點「擴充功能」→「Apps Script」，把 `Code.gs` 的內容整份貼進去（取代預設內容），儲存。

### 3. 部署為網頁應用程式
右上角「部署」→「新增部署作業」：
- 類型：**網頁應用程式**
- 執行身分：**我**
- 誰可以存取：**知道連結的任何人**

部署後會拿到一個網址，格式類似：
`https://script.google.com/macros/s/xxxxxxxx/exec`

### 4. 把網址貼進 App
打開 `index.html`（或部署到 GitHub Pages 後的網址），在「記帳 API 網址」欄位貼上剛拿到的網址，輸入你的名字，就能開始記帳。第一次執行 Apps Script 時 Google 會跳出授權視窗，照著按「允許」即可。

> 沒有網址也沒關係，App 會先切換成離線模式並明確提醒你「這樣記的帳只會存在這台裝置」，之後隨時可以到「設定」補上網址——一補上，尼尼會自動把離線期間記的帳整批同步上去，不用重新輸入一次。

> ⚠️ 如果你之後又回去修改了 `Code.gs` 的內容，記得要「部署」→「管理部署作業」→ 選現有部署 → 按編輯（鉛筆圖示）→ 版本選「新版本」→部署，網址才會套用新程式碼，光按存檔是不會生效的。

### 5.（選用）部署到 GitHub Pages
把 `index.html`、`manifest.json`、`icon-180.png`、`icon-512.png` 一起上傳到一個 GitHub repo，開啟 Settings → Pages，選擇分支後就會拿到一個公開網址。

### 6. 邀請旅伴一起加入
在 App 的「設定」頁貼好 API 網址並儲存後，會出現「生成邀請文字」按鈕，按下去會產生一段包含**專屬邀請連結**的文字，直接複製貼到群組。旅伴點開連結後，記帳 API 網址會自動幫他們填好，只需要輸入自己的名字就能加入同一本帳，不用自己跑一次部署流程。

## 功能

- 貼連結即用，不需登入
- 「設定」頁可以把帳本、旅伴名單、還款紀錄各自匯出成 CSV 檔（含 BOM，Excel/Numbers/Google 試算表打開中文不會亂碼），當作備份或另外分析用
- 一鍵生成邀請文字＋專屬連結，旅伴點開就自動連上同一本帳，只要選自己是誰
- 新增支出精靈（付款人 → 金額幣別 → 類別 → 分帳對象 → 分帳方式），送出後按鈕會鎖住並顯示「記錄中」，不怕手滑連點兩次記成兩筆
- 支援平均分攤 / 按比例（％）/ 自訂金額三種分帳法，自訂金額會即時驗證加總是否等於總額
- 每筆支出可以標分類（餐飲／交通／住宿／門票／購物／其他）
- 帳本依日期分組顯示、可依分類篩選，每筆支出左側有大大的日期方塊；向左滑動可快速編輯/刪除，點開則列出「誰付款」跟每個人要分擔多少
- 每一筆分帳可以獨立標記「已還款」（例如吃完飯當下就先還錢），方便隨手記錄；「總覽」頁的最終結算會自動排除已經還清的部分，不會重複算
- 「總覽」頁（原「結算」）呈現更直覺：旅行總覽卡片（總支出/筆數/平均每人/平均每天）、每人墊付 vs 應負擔長條圖、分類圓餅圖、每日花費趨勢圖，還有最終的建議還款清單（最少轉帳次數）
- 多幣別旅程會分開計算，互不混淆
- 一鍵生成可直接貼到 LINE 的結算摘要文字
- 旅伴管理（新增／改名／刪除，含臨時旅伴）
- 貼上 API 網址後會自動抓這本帳目前的旅伴名單，用下拉選單選「你是哪一位」就好，不用打字、也不怕打錯字；名單裡沒有你的話再選「➕ 我是新旅伴」；「設定」頁也是用同樣的下拉選單切換身分（真的要改名字則到「旅伴」頁）
- 用同樣的名字加入時，系統會找到 Sheet 裡既有的同名旅伴並詢問是否為同一人，確認後就會沿用同一個身分，換裝置也不會變成兩個人
- 離線模式：沒有 API 網址時全部功能都能在本機使用，App 會明確提醒「目前只存在這台裝置」；之後補上網址會自動把離線資料整批同步上去
- 開機/同步只打一次 API（合併成單一 getAll 請求），比逐一請求快很多；資料會先用本機快取立刻顯示畫面，網路資料回來再默默更新，沒有快取時會顯示骨架載入動畫
- 自動同步：App 從背景切回前景、或每 60 秒，會在背景默默跟 Google Sheet 對一次帳；帳本/首頁/總覽也支援手勢下拉刷新
- 首頁與總覽頁都有「立即同步」按鈕，可以手動同步；同步失敗時也會明確提示，不會默默失敗
- 後端加了 LockService 寫入鎖（只鎖新增/修改/刪除，讀取不受影響，速度更快），多人同時記帳不會互相蓋掉彼此的資料
- 自訂數字鍵盤：金額、比例、自訂分帳都用大按鍵鍵盤輸入，手機上更好按
- 刪除支出可在 4.5 秒內按「復原」救回來，不怕手滑
- 「加入主畫面」後是全螢幕獨立 App 外觀，不會看到瀏覽器網址列，頂端也不會被系統狀態列截住
- 支援跟隨系統的深色模式
- Google Sheet 裡的資料是給人看的：日期、付款人、分帳對象、類別都是姓名和白話文字，不是一堆代碼（技術欄位放在後面幾欄，App 讀寫用，正常使用不需要碰）
- 尼尼是一隻胖胖的黑白哈士奇，貫穿整個 App 的插圖與空狀態畫面，還會依情境切換表情（帳算清了會閃亮亮、帳本空空的時候會打瞌睡）

## 這次修正/調整

- **重要 bug**：「新增旅伴」完全沒反應，是因為用了瀏覽器原生的 `prompt()` 彈窗，這種彈窗在「加入主畫面」的獨立模式（standalone PWA）下會被系統直接停用、不會顯示。已經全部改成 App 自己畫的輸入視窗，不受此限制
- 首頁改成「總支出」跟「支出筆數」兩個同樣大的數字並排顯示；最近帳目的呈現方式跟帳本統一，日期都用同樣的大方塊圖示放在最前面
- 「總覽」頁拿掉平均每人、平均每天、每日花費趨勢；每人的款項狀態改成一句話＋顏色卡片（例如「Ben 還要付 300 TWD」），並在最上面加一個整體狀態橫幅（全部結清 / 還有 N 筆待處理），建議還款也加上大頭貼方便一眼辨識

## 之後可以再加的東西（給你參考）

- **多趟旅行切換**：目前一個 App 綁一個 Sheet，若你們常跑很多趟，可以做「旅程列表」讓使用者快速切換
- **每人專屬顏色/大頭貼**：目前用姓名縮寫自動配色，之後可以讓每人自己選顏色或大頭貼
- **收據拍照**：新增支出時附一張照片，之後回頭對帳更方便（Google Sheet 不適合存圖片，需要搭配 Google Drive）
- **簡單通關密語**：目前只要拿到 API 網址就能新增/刪除資料，若在意安全性可以加一組共用 PIN 做基本驗證
- **語音輸入金額備註**：在路上不方便打字時用得到

這些都不影響現在能不能用，先能上線記帳最重要，之後想加哪個都可以再跟我說。
